# Debate Framing

Use this framing when forwarding architect plans to Codex for critique:

1. Restate the plan scope in concrete terms.
2. Ask Codex to challenge assumptions and identify missing operational risks.
3. Require codebase-grounded feedback with file references.
4. Ask for simplifications where implementation is over-engineered.
5. Request a clear agree/disagree conclusion with rationale.
